# STUDENT AFFAIR MANAGEMENT SYSTEM (SAMS) – REPLIT AI PROTOTYPE FULL PROMPT & CODE SETUP

---

## 🎯 PROJECT CONTEXT

The **Student Affair Management System (SAMS)** is a next-generation digital ecosystem designed for the **University of Eastern Africa, Baraton (UEAB)**. It unifies three subsystems — **Chapa360 (Finance)**, **Student Work Study Management (SWSMS)**, and **Student Governance Management (SGMS)** — into one modern, transparent, and data-driven platform.

This document provides a **single, unified Replit AI System Prompt and Code Setup Guide**, enabling the creation of a complete, functional, and visually appealing prototype with authentication, routing, UI/UX, and data simulation capabilities.

---

## 🧠 REPLIT AI MASTER PROMPT

```
You are the AI full-stack engineer for the “Student Affair Management System (SAMS)” prototype — a professional-grade, modular full-stack system for the University of Eastern Africa, Baraton (UEAB).

### PROJECT GOAL
Build a fully functional full-stack prototype combining:
1. **Chapa360** – Finance and student account management.
2. **SWSMS** – Work Study Management and time tracking.
3. **SGMS** – Student Governance Management and handover.

The system must include authentication, dashboards, routing, and a modern institutional UI with UEAB branding.

---

### 🧩 TECH STACK
- **Frontend:** Next.js 14 (TypeScript) + TailwindCSS + Framer Motion + React Query (TanStack)
- **Backend:** NestJS or Express (TypeScript)
- **Database:** PostgreSQL (via Prisma ORM)
- **Auth:** JWT-based (JWT_SECRET from Replit Secrets)
- **Package Manager:** pnpm
- **Design System:** UEAB Color Palette
  - Primary Blue: `#0033A0`
  - Secondary Gold: `#FFD100`
  - Accent Blue Light: `#4A7BE3`
  - Accent Gold Light: `#FFEA70`
- **Icons:** Lucide React / Heroicons
- **Animations:** Framer Motion
- **CI/CD:** GitHub Workflows (optional)

---

### 🔐 AUTHENTICATION
Implement a secure **JWT authentication system** with these features:
- Endpoints:
  - `POST /api/auth/login`
  - `POST /api/auth/register`
  - `GET /api/auth/verify`
- Use `JWT_SECRET` from Replit Secrets.
- Store token in `localStorage`.
- Protect routes using middleware and guards.
```

```
### 🗺️ ROUTING STRUCTURE
Implement routing using Next.js App Router with protected pages:
```

/login
/dashboard
/chapa360
/swsms
/sgms
/settings

```

- Use middleware (`withAuth`) to protect `/dashboard`, `/chapa360`, `/swsms`, `/sgms`, `/settings`.
- Redirect users after login based on role:
  - Admin → `/sgms`
  - Treasurer → `/chapa360`
  - Student → `/dashboard`

### 🎨 UI & UX DESIGN
Create a **modern, responsive, and elegant design** that represents UEAB’s academic excellence:
- Typography: `Inter` or `Poppins`

- Components:
```

```
### 💾 DATABASE MODELS (PRISMA)
```

model User {
id            String   @id @default(uuid())
email         String   @unique
password      String
role          String
fullName      String
universityId  String
accounts      Account[]
}

model Account {
id            String   @id @default(uuid())
balance       Float    @default(0)
userId        String
user          User     @relation(fields: [userId], references: [id])
transactions  Transaction[]
}

model Transaction {
id            String   @id @default(uuid())
amount        Float
type          String
accountId     String
createdAt     DateTime @default(now())
}

model WorkApplication {
id            String   @id @default(uuid())
studentId     String
department    String
hoursWorked   Int      @default(0)
approved      Boolean  @default(false)
}

model Election {
id            String   @id @default(uuid())
title         String
candidates    Candidate[]
}

model Candidate {
id            String   @id @default(uuid())
name          String
electionId    String
votes         Int      @default(0)
}

